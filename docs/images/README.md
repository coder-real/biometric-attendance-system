# Screenshots Guide

## Required Screenshots

To complete your README, take the following screenshots and save them in this folder:

### 1. `system-overview.png`
- **What**: Full dashboard view showing all components
- **How**: Open your dashboard, make sure there's data visible
- **Size**: 1920x1080 recommended

### 2. `dashboard.png`
- **What**: Main dashboard with statistics cards
- **How**: Capture the home page with attendance stats
- **Focus**: Show the analytics cards and recent activity

### 3. `portal.png`
- **What**: Fingerprint portal in action
- **How**: Navigate to `/portal`, wait for GPS lock
- **Focus**: Show the GPS status badge and authentication button

### 4. `attendance-log.png`
- **What**: Live attendance table with location names
- **How**: Go to attendance log page with several records
- **Focus**: Show the location names and "View on Map" buttons

### 5. `students.png`
- **What**: Student management page
- **How**: Open students page with enrolled students
- **Focus**: Show the student list and action buttons

### 6. `sessions.png`
- **What**: Session management interface
- **How**: Navigate to sessions page
- **Focus**: Show active and past sessions

### 7. `wiring-diagram.png` (Optional)
- **What**: Circuit diagram or photo of your hardware setup
- **How**: Use Fritzing or take a clear photo of your breadboard
- **Focus**: Show all connections clearly labeled

## Screenshot Tips

1. **Use Full Screen**: Maximize your browser for clean screenshots
2. **Hide Personal Data**: Blur any sensitive student information
3. **Good Lighting**: If photographing hardware, use good lighting
4. **Consistent Size**: Try to keep all screenshots at similar resolutions
5. **Dark Mode**: Consider using dark mode for modern look

## Tools

- **Windows**: Win + Shift + S (Snipping Tool)
- **Mac**: Cmd + Shift + 4
- **Browser Extension**: Awesome Screenshot, Fireshot
- **Editing**: Paint, GIMP, Photoshop

## After Adding Screenshots

1. Save images in this folder (`docs/images/`)
2. Verify all image paths in README.md work
3. Commit and push to GitHub
4. Check that images display correctly on GitHub
